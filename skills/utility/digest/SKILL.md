# 문서 소화 (Digest)

## 개요

어떤 긴 문서든 받아서 목적·배경에 맞게 이해시켜주는 스킬.
특정 문서 형식을 가정하지 않고 내용을 읽어 스스로 파악한다.

## 사용 시점

- "이거 설명해줘", "요약해줘", "이게 뭔 소리야", "모르겠어"
- 긴 마크다운 문서, 보고서, 스펙, 분석 결과를 처음 접할 때
- `/digest` 직접 호출

## 워크플로우

### Step 1: 사용자 인터뷰

문서를 바로 분석하기 전에 AskUserQuestion으로 한 번에 묻는다.

```yaml
questions:
  - question: "이 문서를 왜 보고 있나요?"
    header: "목적"
    options:
      - label: "전체 파악"
        description: "처음 접하는 문서라 전반적으로 이해하고 싶음"
      - label: "의사결정"
        description: "이 문서 기반으로 무언가 결정해야 함"
      - label: "리뷰 중"
        description: "검토하거나 피드백을 줘야 함"
      - label: "레퍼런스"
        description: "특정 내용을 찾거나 확인하고 싶음"

  - question: "특히 궁금한 부분이 있나요?"
    header: "초점"
    options:
      - label: "없음, 전체"
        description: "문서 전반을 균등하게 설명해줘"
      - label: "결론/결과"
        description: "최종 결론, 판단, 권고사항만"
      - label: "위험/문제"
        description: "이슈, 리스크, 주의사항 중심"
      - label: "직접 지정"
        description: "특정 섹션이나 키워드 (Other로 입력)"

  - question: "설명을 얼마나 자세히 해드릴까요?"
    header: "깊이"
    options:
      - label: "알아서 판단 (Recommended)"
        description: "문서 길이와 복잡도에 따라 자동 조절"
      - label: "핵심만 짧게"
        description: "3-5줄 요약으로 충분"
      - label: "꼼꼼하게"
        description: "섹션별 상세 해설"
```

### Step 2: 문서 파악

문서 내용을 읽고 다음을 파악한다. 타입을 사전 분류하지 말고 내용에서 직접 추론한다.

- **이 문서의 목적**: 무엇을 위해 만들어진 문서인가
- **핵심 결론의 위치**: 어디에 최종 판단/결과가 있는가
- **전문 용어 밀도**: 해설이 얼마나 필요한가
- **독자 가정**: 이 문서가 어떤 독자를 전제하고 있는가

**요약 깊이 자동 판단** (사용자가 "알아서 판단" 선택 시):

| 조건 | 깊이 |
|------|------|
| ~100줄 또는 전문 용어 적음 | 결론만 (3-5줄) |
| 100-250줄 또는 전문 용어 중간 | 결론 + 섹션 요약 |
| 250줄+ 또는 전문 용어 많음 | 결론 + 섹션 해설 + 용어 풀이 |

사용자가 "초점"을 지정했으면 해당 부분에 집중한다.

### Step 3: 출력

```markdown
## TL;DR
[사용자의 목적 기준으로 재구성한 핵심 결론 — 2-5줄]

## 해설
[섹션별 자연어 설명. 전문 용어는 괄호 안에 풀어서 표기.
깊이는 Step 2에서 판단한 수준으로]

## 더 궁금할 만한 것
1. [문서 내용 + 사용자 목적에서 추론한 예상 질문]
2. ...
3. ...

→ 번호로 선택하거나 직접 질문하세요
```

### Step 4: Q&A 루프

사용자가 번호를 선택하거나 질문하면 해당 내용에 집중해서 답한다.
답변 후 명시적으로 종료 여부를 확인한다:

```yaml
AskUserQuestion:
  question: "다른 궁금한 점이 있으신가요?"
  header: "Q&A"
  options:
    - label: "추가 질문 있음"
      description: "다른 내용이 궁금합니다 (번호 또는 직접 입력)"
    - label: "충분히 이해했음"
      description: "설명이 도움이 되었습니다, 종료합니다"
```

"추가 질문 있음" → 질문/번호를 입력받아 답변 후 Step 4 반복.
"충분히 이해했음" → 간단한 마무리 메시지 후 종료.
