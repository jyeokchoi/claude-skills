Slack 채널들에서 이번 주의 크로스팀 인사이트를 수집하고 요약하여 Slack에 게시하는 스킬.

## 입력 파싱

$ARGUMENTS를 파싱한다:

1. **채널 프리픽스** (필수): 검색할 채널 프리픽스 (예: `vrew_`, `design_`, `platform_`)
   - 프리픽스가 없으면 중단하고 출력: "채널 프리픽스를 입력해주세요. 예: /weekly-digest vrew_"

2. **--post-to** (선택): 요약을 게시할 채널 이름. 기본값: 프리픽스에서 trailing `_`를 제거한 이름 (예: `vrew_` → `#vrew`, `design_` → `#design`)

3. **--week** (선택): 주의 시작 날짜 (월요일). 기본값: 이번 주 월요일.

4. **--exclude** (선택): 추가로 제외할 채널 키워드 (쉼표 구분). 기본 제외 목록에 추가된다.

## 1단계: 채널 목록 수집

`slack_search_channels`로 프리픽스에 해당하는 모든 공개 채널을 검색한다.
- `include_archived: false`로 아카이브된 채널 제외
- 페이지네이션하여 모든 채널 수집

### 채널 필터링

봇/알림/자동화 채널을 제외한다. 다음 키워드가 포함된 채널은 스킵:
- `알림`, `이슈`, `앱리뷰`, `bot`, `qabot`, `chatbot`, `에러`, `사용량`, `노션`, `lokalise`, `피드백`, `reporting`
- `--exclude`로 전달된 추가 키워드도 적용

## 2단계: 채널 메시지 읽기

남은 채널들의 이번 주 메시지를 병렬로 읽는다.
- `oldest` 파라미터에 해당 주 월요일 00:00 KST(UTC+9)의 Unix timestamp 사용
- `response_format: "concise"`로 읽기
- 한 번에 최대 10개 채널씩 병렬 호출
- 활동이 없는 채널은 건너뛴다

큰 채널 데이터가 파일로 저장된 경우 Bash로 앞부분을 추출하여 읽는다.

## 3단계: 크로스팀 인사이트 필터링

**핵심 원칙: "이 팀이 아닌 다른 팀의 사람이 이걸 알면 행동이 달라지거나, 궁금해하거나, 시너지를 낼 수 있는가?"**

### 포함 기준 (다른 팀에 유용)
- 사용자 행동 데이터/분석 — 방법론이나 인사이트를 다른 제품에 적용 가능
- 시장/고객 반응 — 기술 투자 방향성, 시장 수요 신호
- UX 교훈 — 전사적으로 적용 가능한 디자인/인터랙션 인사이트
- AI/LLM 전략 변경 — 비용 최적화, 벤더 변경 등 다른 팀 참고 가능
- B2B/법무/컴플라이언스 — 전사적 대응 필요
- 비즈니스 모델 성과 — 어필리에이트, 파트너십 등 검증된 모델

### 제외 기준 (팀 내부 관심사)
- RC/QA 버그 리포트, 핫픽스 상세
- 개인별 체크인/작업 현황
- 내부 인프라 업그레이드 (Kube, 서버 등) — 다른 팀에 직접 영향 없는 경우
- 내부 기능 개발 진행 상황
- 코드 리뷰 리마인더, PR 목록
- 봇/자동화 메시지

## 4단계: 요약 작성

수집된 인사이트를 파일(`~/.claude/projects/{project}/weekly-cross-team-summary.md`)에 정리한다.

각 항목에 포함할 정보:
- 카테고리 이모지 + 볼드 제목
- 출처 채널, 작성자, 날짜
- 채널 링크 (Slack workspace URL + `/archives/{channel_id}`)
- 2~3줄 요약
- `→ _시사점_` 형태의 크로스팀 적용 포인트

## 5단계: Slack 게시

**`slack-write` 스킬을 로드하여 포맷 규칙과 전송 워크플로우를 따른다.**

### 메시지 구조

**본문** (요약 — 5,000자 이내):
```
📊 *{프리픽스} 주간 크로스팀 인사이트 ({날짜 범위})*

{한 줄 설명}
상세는 🧵 스레드에.

*{이모지} {카테고리 1}*
• {인사이트 1줄 요약}
• {인사이트 1줄 요약}

*{이모지} {카테고리 2}*
• ...

---
🤖 _Written by Claude Code_
```

**스레드 댓글** (상세 — 각 5,000자 이내):
```
🧵 *상세 내용*

*{번호}. {제목}*
_{작성자} · #{채널} · {날짜}_
{채널 링크}
{2~3줄 상세 설명}
→ _{크로스팀 시사점}_

...

---
🤖 _Written by Claude Code_
```

### 전송 순서
1. 초안을 사용자에게 보여준다
2. 사용자 승인 후 본문 전송
3. 본문의 `message_ts`를 사용하여 스레드 댓글 전송
4. 상세가 5,000자를 초과하면 여러 스레드 댓글로 분할

### 필수 설정
- `content_type: "text/plain"` 사용
- `slack-write`의 실수 방지 체크리스트 모두 통과

**사용자의 명시적 승인 없이는 절대 Slack에 게시하지 않는다.**

이제 실행하라.
